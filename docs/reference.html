LANGUAGE REFERENCE

This is the complete word list for Bare Naked Forth. Kernel primitives are implemented in 6809 assembly and are always available. Library words are written in Forth and loaded with INCLUDE. Compiler directives are handled by the cross-compiler (fc.py) and do not generate runtime code.

Kernel Primitives

These words are implemented in 6809 machine code in kernel.asm. They are always available to every program.

Threading
Word Stack Description
DOCOL — Enter a colon definition. Pushes IP to return stack, sets IP to word body. Generated automatically by the compiler for : definitions.
DOVAR ( -- addr ) Enter a variable. Pushes the address of the variable's data cell. Generated automatically by the compiler for VARIABLE definitions.
EXIT — Return from a colon definition. Pops IP from the return stack. The compiler emits this automatically at the end of every : definition.
LIT ( -- n ) Push the next cell in the thread as a 16-bit literal. The compiler emits this before every numeric constant in source code.
Stack
Word Stack Description
DUP ( n -- n n ) Duplicate the top of stack.
DROP ( n -- ) Discard the top of stack.
SWAP ( n1 n2 -- n2 n1 ) Exchange the top two stack items.
OVER ( n1 n2 -- n1 n2 n1 ) Copy the second item to the top of the stack.
?DUP ( x -- x x | 0 ) Duplicate only if non-zero. If the top of stack is zero, leave it unchanged.
2DUP ( n1 n2 -- n1 n2 n1 n2 ) Duplicate the top pair of stack items.
2DROP ( n1 n2 -- ) Discard the top two stack items.
ROT ( n1 n2 n3 -- n2 n3 n1 ) Rotate the third item to the top of the stack.
PICK ( ...xn ... x0 n -- ...xn ... x0 xn ) Copy the nth stack item to the top. 0 PICK is DUP, 1 PICK is OVER.
Arithmetic
Word Stack Description
+ ( n1 n2 -- sum ) Add the top two stack items.
- ( n1 n2 -- diff ) Subtract. Result is n1 − n2.
* ( n1 n2 -- product ) Multiply. Uses the 6809's hardware MUL instruction.
/MOD ( n1 n2 -- rem quot ) Divide n1 by n2. Pushes remainder, then quotient.
NEGATE ( n -- -n ) Two's complement negate. Flips the sign of the top of stack.
ABS ( n -- |n| ) Absolute value.
MIN ( n1 n2 -- min ) Signed minimum of the top two stack items.
MAX ( n1 n2 -- max ) Signed maximum of the top two stack items.
0MIN ( n -- min(n,0) ) Clamp to maximum 0. Returns n if negative, else 0.
0MAX ( n -- max(n,0) ) Clamp to minimum 0. Returns n if positive, else 0.
2* ( n -- n*2 ) Arithmetic left shift by 1 (multiply by 2).
2/ ( n -- n/2 ) Arithmetic right shift by 1 (divide by 2, sign-preserving).
MDIST ( addr1 addr2 -- d ) Manhattan distance between two (x,y) byte pairs at addr1 and addr2. Returns |x1−x2| + |y1−y2|.
Comparison
Word Stack Description
= ( n1 n2 -- flag ) True ($FFFF) if n1 equals n2, false ($0000) otherwise.
<> ( n1 n2 -- flag ) True if n1 does not equal n2.
< ( n1 n2 -- flag ) True if n1 is less than n2 (signed comparison).
> ( n1 n2 -- flag ) True if n1 is greater than n2 (signed comparison).
0= ( n -- flag ) True if n is zero.
U< ( u1 u2 -- flag ) True if u1 is less than u2 (unsigned comparison). Use this for address / size comparisons where signed < would give wrong results near $8000.
WITHIN ( n lo hi -- flag ) True if lo ≤ n < hi (unsigned, half-open range).
Memory
Word Stack Description
@ ( addr -- n ) Fetch a 16-bit value from memory address.
! ( n addr -- ) Store a 16-bit value to memory address.
C@ ( addr -- byte ) Fetch an 8-bit byte from memory address.
C! ( byte addr -- ) Store an 8-bit byte to memory address.
FILL ( addr count byte -- ) Fill count bytes starting at addr with byte.
CMOVE ( src dest count -- ) Copy count bytes from src to dest. Ascending copy (safe when dest > src).
+! ( n addr -- ) Add n to the 16-bit cell at addr. Equivalent to DUP @ ROT + SWAP ! but faster.
Logic / Bitwise
Word Stack Description
AND ( n1 n2 -- n3 ) Bitwise AND of the top two stack items.
OR ( n1 n2 -- n3 ) Bitwise OR of the top two stack items.
LSHIFT ( n count -- n' ) Logical left shift n by count bits.
RSHIFT ( n count -- n' ) Logical right shift n by count bits.
XOR ( n1 n2 -- n3 ) Bitwise exclusive-or of the top two stack items.
INVERT ( n -- ~n ) Bitwise complement. Equivalent to -1 XOR.
I/O
Word Stack Description
EMIT ( c -- ) Write character to video RAM at the cursor position and advance the cursor. Converts ASCII to VDG encoding.
CR ( -- ) Advance the cursor to the start of the next row.
KEY ( -- c ) Wait for a keypress with debounce and auto-repeat. Push the ASCII code. Blocking.
KEY? ( -- c | 0 ) Non-blocking key scan. Returns the ASCII code of any pressed key, or 0 if no key is held.
KBD-SCAN ( col-mask -- row-bits ) Raw keyboard matrix scan. Writes col-mask to PIA0-B ($FF02) and returns the row bits. Used by the keyboard library for direct key testing.
AT ( row col -- ) Set the cursor to the given row (0–15) and column (0–31).
TYPE ( addr len -- ) Print len bytes starting at addr via EMIT. Used by S" ... " and .".
COUNT ( c-addr -- addr+1 len ) Extract a counted string: c-addr points at a length byte; returns the address of the first character plus the length ready for TYPE.
Control Flow
Word Stack Description
0BRANCH ( flag -- ) Branch forward by the inline offset if flag is zero. Used by the compiler to implement IF and UNTIL.
BRANCH ( -- ) Unconditional branch by the inline offset. Used by the compiler for ELSE, AGAIN, and LOOP.
DO ( limit index -- ) Begin a counted loop. Pushes limit and index to the return stack.
LOOP — Increment the loop index by 1. If index < limit, branch back to the word after DO.
I ( -- n ) Push the current loop index onto the data stack.
+LOOP ( n -- ) Increment the loop index by n (which may be negative). If the index has not crossed the limit, branch back to the word after DO.
J ( -- n ) Push the index of the outer enclosing DO loop (for nested loops). I is the inner index.
UNLOOP ( -- ) R:( limit index -- ) Discard the current loop's index/limit from the return stack. Required before EXIT inside a DO ... LOOP to keep the return stack balanced.
Return Stack
Word Stack Description
>R ( n -- ) R:( -- n ) Move the top of the data stack to the return stack. Must be balanced with R> before the word returns.
R> ( -- n ) R:( n -- ) Move the top of the return stack to the data stack.
R@ ( -- n ) R:( n -- n ) Copy the top of the return stack to the data stack without removing it.
Screen Timing
Word Stack Description
VSYNC ( -- ) Block until the next vertical sync (start of vertical blanking). Polls PIA0 HSYNC/VSYNC bits. Use to pace animation at 60 Hz.
WAIT-PAST-ROW ( row -- ) Block until the scanning beam has passed the given pixel row (0–191). Used for beam-chasing: split-frame updates that run above or below the beam without tearing.
COUNT-BLANKING ( -- cycles ) Return the number of inner-loop cycles counted during the most recent vertical blanking interval. Used to tune beam-chasing offsets empirically.
Graphics

RG6 (256×192 NTSC artifact-color) pixel and sprite primitives. The library forth/lib/rg-pixel.fs sets up tables and VRAM; these words then run at native 6809 speed in the kernel.

Word Stack Description
RG-PSET ( x y color -- ) Plot an artifact-color pixel. x is 0–127 (pixel pairs), y is 0–191. color: 0=black, 1=blue, 2=red, 3=white.
RG-LINE ( x1 y1 x2 y2 color -- ) Draw a line using Bresenham's algorithm. Clips at screen edges.
RG-CHAR ( char cx cy -- ) Render one glyph from the kernel font at character cell (cx,cy). Uses the font base, glyph size, and row metrics configured in kernel variables at $0075–$007B.
SPR-DRAW ( addr x y -- ) Draw a sprite at pixel (x,y). Sprite format: width byte, height byte, then 2-bits-per-pixel packed rows. Color 0 is transparent.
SPR-ERASE-BOX ( addr x y -- ) Erase the bounding box of a sprite at (x,y) to black. Sister to SPR-DRAW.
Beam

Beam primitives save a line's original pixels into a buffer, draw a colored slice over them, then restore them later. Used for maser beams, laser lines, and any transient straight-line drawing that must not damage the background. Obstacle-finding supports collision detection.

Word Stack Description
BEAM-TRACE ( x1 y1 x2 y2 buf -- count ) Trace the pixels along the line from (x1,y1) to (x2,y2). Save each pixel's original color into buf. Return the pixel count.
BEAM-DRAW-SLICE ( buf start count color -- ) Draw count pixels of a previously-traced beam in the given color, starting at offset start in buf.
BEAM-RESTORE-SLICE ( buf start count -- ) Restore count pixels of a traced beam to their original colors, starting at offset start.
BEAM-FIND-OBSTACLE ( buf count -- index|-1 ) Scan a beam's save buffer for the first non-black original pixel. Returns the index of the obstacle, or −1 if none. Used for collision detection.
BEAM-SCRUB-POS ( buf index -- ) Overwrite one saved pixel at index with black ($00). Used to “forget” a pixel so a later restore will leave it as background, e.g. after an object on the beam path moves.
Spatial
Word Stack Description
PROX-SCAN ( cx cy r arr n -- mask ) Proximity scan. Tests each (x,y) pair in arr (up to n entries) against center point (cx,cy) with radius r. Returns a bitmask of hits.
Data
Word Stack Description
sprite-data ( -- addr ) Address of kernel sprite data (5 sprites × 12 bytes). Copy to application sprite region with CMOVE.
font-data ( -- addr ) Address of kernel font data (59 glyphs × 8 bytes). Copy to $9000 with CMOVE.
Constants
Name Value Description
KERN-VERSION $0100 (1.0) Kernel version in packed BCD: high byte is the major version, low byte is the minor (so $0100 means 1.0). Exported from KERN_VERSION in kernel.asm via the kernel map. Use it to gate features or print the version, e.g. hello.fs prints it on boot.
System
Word Stack Description
HALT — Spin forever (BRA *). Marks the end of the application.

Compiler Directives

These are handled by fc.py at compile time. They do not produce runtime code — they control how code is generated.

Directive Syntax Description
: : name ... ; Begin a colon definition. Compiles the body as a threaded code sequence with DOCOL header and EXIT tail.
; End a colon definition. Emits EXIT.
VARIABLE VARIABLE name Create a named variable. Allocates a 2-byte data cell initialized to zero. Pushes the cell address when called.
CONSTANT n CONSTANT name Create a named constant. name compiles as LIT n.
CHAR CHAR c Push the ASCII value of the next character. CHAR A compiles as LIT 65.
IF IF ... THEN
IF ... ELSE ... THEN		 Conditional execution. Pops flag; if zero, skips to ELSE or THEN.
ELSE Begin the false branch of an IF.
THEN End an IF...ELSE...THEN block.
DO ... LOOP limit index DO ... LOOP Counted loop. Iterates from index to limit−1. Use I inside the loop to read the current index.
BEGIN ... AGAIN BEGIN ... AGAIN Infinite loop. The body repeats forever.
BEGIN ... UNTIL BEGIN ... flag UNTIL Loop with exit condition. Repeats while flag is zero; exits when flag is non-zero.
INCLUDE INCLUDE path Splice the contents of another Forth file at this point in the source. Paths are relative to the including file. Nested includes are supported.
S" S" text" Compile a string literal. At runtime, pushes the address and length of the string onto the stack.
." ." text" Print a string literal. Compiles as S" followed by TYPE.
\ (backslash) \ comment text Line comment. Everything after \ to end of line is ignored.
( ) ( comment text ) Block comment. Text between ( and ) is ignored. May span multiple lines.
CODE CODE name
  ... 6809 asm ...
;CODE		 Define a word implemented in inline 6809 assembly. The body is assembled by lwasm and placed in app space. End the assembly with ;NEXT (the macro for JMP [,X++]) to return to the threaded interpreter. Use this for performance-critical inner loops.
;CODE End a CODE block.
KCODE KCODE name
  ... 6809 asm ...
;KCODE		 Like CODE, but the assembled bytes are packed into kernel space (appended to the kernel region, not the app). Use this when app space is tight but kernel headroom is available, or when several apps share a primitive. Same rules as CODE: end with ;NEXT. fc.py assembles KCODE twice — once to get sizes, then again with final variable addresses patched in.
;KCODE End a KCODE block.

Numeric literals can be decimal (42) or hexadecimal with a $ prefix ($FF02). Negative numbers work too: -1 compiles as $FFFF.

Accessing variables from inline assembly

Inside CODE and KCODE blocks, fc.py exposes variable addresses as EQU labels so you can load and store directly without going through the data stack:

Label form What it resolves to
FVAR_name Address of a Forth VARIABLE you defined in your own source. fc.py walks every VARIABLE in the program, assigns addresses in app space, and injects FVAR_name EQU $xxxx before each assembly pass. Use LDD FVAR_score / STD FVAR_score etc. to read and write app variables from assembly.
KVAR-name Address of a kernel variable (the VAR_* symbols in kernel.asm, e.g. cursor, sprite scratch, beam scratch). fc.py reads these out of kernel.map and exposes them as Forth CONSTANTs with a KVAR- prefix. For example, KVAR-CUR resolves to the address of VAR_CUR. Use them to read or adjust kernel state from application Forth or from inline assembly.

The two systems work together: an app defines its own variables with VARIABLE and reaches them from assembly as FVAR_…; and it reaches shared kernel state as KVAR-…. Example pattern from forth/lib/rg-pixel.fs, where the app caches the kernel VRAM pointer:

$0600 DUP rv !  KVAR-RGVRAM !   \ set local and kernel copies

CODE fast-plot   ( x y -- )
    LDD  FVAR_rv       \ app variable: VRAM base
    ADDD #0            \ ... computation ...
    ;NEXT
;CODE

Library Words

Library words are written in Forth and live in forth/lib/. Load them with INCLUDE at the top of your source file. Some libraries depend on others — the required include order is shown below.

A few words grouped here by topic — VSYNC, WAIT-PAST-ROW, COUNT-BLANKING, RG-PSET, RG-LINE, RG-CHAR, SPR-DRAW, SPR-ERASE-BOX, and the BEAM-* family — are actually kernel primitives, listed above in the Kernel Primitives section. They live in the kernel for speed, but the initialization patterns and usage context belong with their functional library, so we describe them here too.

Keyboard Matrix

INCLUDE lib/keyboard.fs

Word Stack Description
KEY-HELD? ( col row -- flag ) Non-blocking key test. Returns non-zero if the specified key is currently pressed. Does not affect KEY debounce state.

Each key has its own column. Keys that appear to be in a “group” (arrows, digits) actually share a row bit, not a column. Source of truth: kernel.asm KEY_TABLE.

Column constants — each selects one column of the keyboard matrix:

Constant Value Keys (rows 0–6)
KB-C0$FE@ H P X 0 8 ENT
KB-C1$FDA I Q Y 1 9 CLR
KB-C2$FBB J R Z 2 : BRK
KB-C3$F7C K S UP 3 ; ALT
KB-C4$EFD L T DN 4 , CTL
KB-C5$DFE M U LT 5 - SHF
KB-C6$BFF N V RT 6 . F1
KB-C7$7FG O W SP 7 / F2

Row constants — select one row across all columns:

Constant Value Keys across columns 0–7
KB-R0$01@ A B C D E F G
KB-R1$02H I J K L M N O
KB-R2$04P Q R S T U V W
KB-R3$08X Y Z UP DN LT RT SP — all arrows share this row
KB-R4$100 1 2 3 4 5 6 7 — all digits share this row
KB-R5$208 9 : ; , - . /
KB-R6$40ENT CLR BRK ALT CTL SHF F1 F2
KB-R7$80(joystick comparator — not a keyboard row)

Example: test if the UP arrow is held:

KB-C3 KB-R3 KEY-HELD?

Memory Map

Default layout (text mode, tutorials):

Address Contents
$0050–$0082 Kernel variables (cursor, key state, line/sprite scratch, text config)
$0400–$05FF Video RAM — 32 columns × 16 rows, VDG alphanumeric mode
$1000 Kernel code begins (DOCOL, DOVAR, CFA table, primitives, START)
$2000 Application load address (APP_BASE) — cross-compiled Forth thread
$6000–$6102 Font data (if init-font is called) — 37 glyphs × 7 bytes
$7E00 Data stack base (U register, grows downward)
$7FFE Return stack init (S register, grows downward)

All-RAM layout (graphics mode):

Address Contents
$0050–$0082 Kernel variables
$0600–$1DFF Video RAM — RG6 mode (256×192, 6144 bytes)
$0E00 Bootstrap (copies staged kernel to $E000, enables all-RAM via STA $FFDF)
$2000 Application code (APP_BASE)
$8000+ Game data region (all-RAM, above ROM shadow)
$DE00 Data stack base (U register, grows downward)
$E000 Kernel code (final location, all-RAM mode) / Return stack init (S, grows down from $DFFF)

ASCII → VDG Encoding

The CoCo's MC6847 VDG uses a 6-bit character code. EMIT handles this conversion automatically, but if you're writing directly to video RAM:

screen_byte = $40 OR ( ascii AND $3F )

Uppercase A–Z need no conversion. Space becomes $60. The VDG has no lowercase characters.
