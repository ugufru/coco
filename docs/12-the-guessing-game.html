THE GUESSING GAME 1

Putting It All Together

This chapter is the capstone. The guessing game uses every tool built since Chapter 1: the stack, defined words, variables, loops, decisions, keyboard input, number printing, cursor positioning. None of it is new — what's new is how it fits together into a real, playable program.

The Design

The CoCo picks a secret number between 1 and 9. You type digits until you find it. Each guess gets a hint: TOO HIGH or TOO LOW. The screen shows your current try count.

Game Loop

show title screen
press any key → pick secret number
loop:
  show GUESS: prompt + try count
  wait for digit key
  if guess = secret → CORRECT! stop
  if guess < secret → TOO LOW
  if guess > secret → TOO HIGH

Picking a Secret Number

The CoCo has no clock, so we use the player's own keypress to seed the random number. When the game shows PRESS ANY KEY, KEY blocks until they press something. Whatever character they press (A, space, ENTER — anything) becomes the seed: 

: PICK
  KEY 31 * 3 + 9 /MOD DROP 1 + SECRET ! ;

Multiply the ASCII code by 31, add 7, take the remainder mod 9, add 1. The result is 1–9. Different keys give different numbers; the same key always gives the same number, but players don't think about that.

THE GUESSING GAME 2

Screen Layout with AT

Each message lives in its own word. AT jumps the cursor to a fixed position; the EMIT calls that follow write text there. The key words: 

: .PROMPT
  5  6 AT
  CHAR G EMIT CHAR U EMIT CHAR E EMIT CHAR S EMIT CHAR S EMIT
  CHAR : EMIT 32 EMIT 32 EMIT ;   \ trailing spaces clear prev digit

: .HI
  5 10 AT
  CHAR T EMIT CHAR O EMIT CHAR O EMIT 32 EMIT
  CHAR H EMIT CHAR I EMIT CHAR G EMIT CHAR H EMIT 32 EMIT 32 EMIT ;

: .LO
  5 10 AT
  CHAR T EMIT CHAR O EMIT CHAR O EMIT 32 EMIT
  CHAR L EMIT CHAR O EMIT CHAR W EMIT 32 EMIT 32 EMIT 32 EMIT ;

.HI and .LO both write to row 10. The trailing spaces in .LO erase the extra letters from .HI ("HIGH" is four letters, "LOW" is three — a space must overwrite the stray H).

The Game Loop

: PLAY
  0 TRIES !
  .TITLE .PRESS
  PICK                     \ wait for keypress; set SECRET
  BEGIN
    .PROMPT .TRIES
    12  6 AT          \ park cursor at input slot
    KEY DIGIT? IF
      DUP EMIT             \ echo the digit
      CHAR 0 -             \ convert ASCII to value
      TRIES @ 1 + TRIES !
      .TRIES
      DUP SECRET @ = IF
        DROP .WIN EXIT     \ correct — show win and return
      THEN
      SECRET @ < IF .LO ELSE .HI THEN
    ELSE
      DROP                 \ ignore non-digit keys
    THEN
  AGAIN ;

DUP SECRET @ = leaves the guess on the stack while it tests equality. If equal: DROP .WIN EXIT — discard the guess, show the win message, and leave PLAY via EXIT. If not equal, the guess is still on the stack for the < comparison that produces the hint.

Do-It-Yourself
  1. Press different keys at the "PRESS ANY KEY" prompt. Do you get different secret numbers? Try A, B, and C.
  2. Add a word .RANGE that prints 1 TO 9 on row 3 at column 10, and call it from .TITLE. Does it help the player?
  3. After .WIN, print the secret number so the player can see it. Where in PLAY do you add this?
  4. Change the range from 1–9 to 1–5 by modifying one number in PICK. Which number changes, and to what?

Patterns

Seed RNG from keypress: KEY 31 * 7 + 9 /MOD DROP 1 +

One word per screen region keeps game logic clean

Trailing spaces in messages erase leftover characters

Techniques

DUP SECRET @ = tests without consuming

EXIT inside BEGIN…AGAIN escapes the loop

Screen layout defined once; loop just calls the words

Concepts

A complete program is many small words working together

Each word does one job and is easy to test alone

The stack carries data between words invisibly
