MAKE YOUR OWN WORDS 1

Words That Listen to the Stack

In the first two chapters, every word you defined did exactly the same thing every time you called it. TITLE always printed COLOR FORTH. DASH always printed a dash. RULE always printed eleven of them.

That's fine for fixed output — but what about words that do different things depending on what's on the stack?

Here's an idea. The alphabet has 26 letters. The letter A is at position 0. B is at position 1. Z is at position 25. What if you wrote a word called LETTER that takes a position off the stack and prints the letter at that position? 

: LETTER  CHAR A + EMIT ;

That's the whole definition. CHAR A pushes 65 (the ASCII code for A). + adds whatever number was on the stack to 65. EMIT prints the result. So:

You write Prints
0 LETTERA
1 LETTERB
7 LETTERH
25 LETTERZ

LETTER is a word with a parameter. Its behavior depends on what you give it. This is the stack at work as a parameter-passing mechanism — and it's one of the things that makes Forth so flexible.

Stack effect of LETTER: ( n -- )
It takes one number and leaves nothing — consuming the position and printing the letter.

Subtraction: Finding the Distance

Illustration A ruler laid across the alphabet — A at one end, Z at the other. A bracket shows the distance from A to O is 14. CoCo character reading the ruler, pencil in hand.

Here's the natural question: if you want to print the letter O, what number do you give LETTER? You could count: A=0, B=1, C=2 … O=14. Or you could ask Forth to figure it out for you using subtraction: 

CHAR O CHAR A -

CHAR O pushes 79. CHAR A pushes 65. - pops both and pushes 79 - 65 = 14. No counting required.

-  ( n1 n2 -- diff )

Pop two numbers, push n1 - n2. The top of the stack is subtracted from the number below it.

10 3 - → 7

CHAR O CHAR A - → 14

So instead of writing 14 LETTER, you can write: 

CHAR O CHAR A - LETTER

This reads almost like English: "the distance from A to O, then print that letter." Both produce exactly the same result — O on screen. Use whichever is clearer to you.

MAKE YOUR OWN WORDS 2

Building a Vocabulary

With LETTER in hand, you can write words that spell out anything — and those words read naturally: 

: SP  32 EMIT ;

: COLOR
  CHAR C CHAR A - LETTER
  CHAR O CHAR A - LETTER
  CHAR L CHAR A - LETTER
  CHAR O CHAR A - LETTER
  CHAR R CHAR A - LETTER ;

: FORTH
  CHAR F CHAR A - LETTER
  CHAR O CHAR A - LETTER
  CHAR R CHAR A - LETTER
  CHAR T CHAR A - LETTER
  CHAR H CHAR A - LETTER ;

: TITLE  COLOR SP FORTH CR ;

TITLE HALT
Your CoCo's screen:
COLOR FORTH
Illustration A pyramid of boxes. Bottom row: LETTER, SP. Middle row: COLOR, FORTH. Top: TITLE. Arrows point upward showing each level built on the one below. CoCo character standing beside it, arms crossed, looking satisfied.

Look at how the program is organized. At the bottom, tiny words do tiny jobs. At the top, TITLE just says what to do in plain terms. This is the Forth way: build a language that talks about your problem, then write the program in that language.

The top-level line — TITLE HALT — reads like an instruction you'd give to a person. You've built yourself a language where "print the title" is a single word.

Words must be defined before they are used. LETTER must appear before COLOR, and both must appear before TITLE. Forth reads your source from top to bottom and only knows about words it has already seen.

What Happens Inside

When your program runs TITLE, the interpreter jumps into the TITLE definition and executes each word inside it. COLOR is a word, so the interpreter jumps into that definition. Inside COLOR, the first thing it encounters is CHAR C — which pushes 67 — then CHAR A which pushes 65, then - which leaves 2, then LETTER which adds 65 and emits character 67: C.

The interpreter keeps a tidy record of where to return after each word using the return stack. As long as you don't write words that call each other in circles (which we'll come back to much later), the return stack handles everything automatically.

The Forth Vocabulary Model

Every program builds on what came before:

  1. Primitive words built into the kernel (EMIT, +, -, CR)
  2. Your utility words built from those (LETTER, SP)
  3. Your domain words built from utilities (COLOR, FORTH)
  4. Your top-level program built from domain words (TITLE)

Each layer speaks at a higher level than the one below it.

MAKE YOUR OWN WORDS 3

Choosing Good Names

The names you choose for your words matter more than you might think. A well-named word makes the program read naturally. A badly-named one forces the reader to stop and think.

Compare these two definitions of the same word: 

: X  CHAR A + EMIT ;       \ hard to read
: LETTER  CHAR A + EMIT ;  \ obvious

When you later write 7 X, the reader wonders: what is X? When you write 7 LETTER, the intent is clear. The code becomes its own documentation.

Forth programmers tend to use short, punchy names for very common things (SP, CR) and descriptive names for domain-specific words (TITLE, LETTER). There are no hard rules — just the goal of readability.

Do-It-Yourself
Illustration CoCo character at a chalkboard with the alphabet written out, pointing at various letters and writing offset numbers beneath them. Chalk dust in the air. Pleased expression.
  1. Use LETTER to spell your first name, one letter at a time. Use CHAR X CHAR A - to find the offset for each letter.
  2. Define a word called DIGIT that works like LETTER but for the digits 0–9: 0 DIGIT prints 0, 7 DIGIT prints 7. Hint: what is the ASCII code for 0?
  3. Write a word called MY-NAME that prints your full name followed by CR. Then write a word called BANNER that calls TITLE, then MY-NAME.
  4. What does CHAR A CHAR A - leave on the stack? What letter does CHAR A CHAR A - LETTER print?
