NUMBERS ON SCREEN 1

The Invisible Number Problem

After Chapter 7 you can compare numbers. After Chapter 8 you can read keys. But there's a gap: you still can't show a computed result. Try this mentally — 6 7 * puts 42 on the stack, but nothing appears on screen. To display it you'd have to somehow decompose 42 into the characters '4' and '2' and emit them. That's exactly what this chapter builds.

* — Multiply

*  ( n1 n2 -- prod )

Pop two values, push their product (low 16 bits).

6 7 *     \ stack: ( 42 )
3 100 *   \ stack: ( 300 )
-3 4 *    \ stack: ( -12 )

Signed and unsigned values mix freely — * gives the correct low 16 bits in all cases.

/MOD — Divide with Remainder

Illustration A pie cut into slices. "17 slices, 5 people: each gets 3, with 2 left over." The quotient (3) and remainder (2) sit on the stack as two separate values. Remainder is below, quotient is on top.

/MOD  ( n1 n2 -- rem quot )

Divide n1 by n2. Push remainder, then quotient (quotient on top).

17 5 /MOD   \ stack: ( 2 3 )  — rem=2, quot=3
10 3 /MOD   \ stack: ( 1 3 )  — rem=1, quot=3
 7 7 /MOD   \ stack: ( 0 1 )  — rem=0, quot=1

Having both results at once is the key: 42 10 /MOD gives 2 (remainder) and 4 (quotient) in a single step, which is exactly what a decimal digit extractor needs.

The Decimal Output Algorithm

To print the number 423, you need to produce three characters — '4', '2', '3' — in that order. But /MOD extracts the last digit first:

Step Quotient Remainder (digit)
423 ÷ 10423
42 ÷ 10 4 2
4 ÷ 10 0 4

The digits come out backwards (3, 2, 4) but we need to print them forwards (4, 2, 3). The trick: recurse before printing. Process the quotient first (which recurses deeper), then print the remainder on the way back out.

NUMBERS ON SCREEN 2

Building the Print Words in Forth

All four words below are defined in Forth — no new machine code needed. Each one builds on what already exists.

NEGATE — flip the sign

: NEGATE   0 SWAP - ;

Pushes 0, swaps it beneath the number, then subtracts: 0 − n = −n. Already works with SWAP and - from Chapter 2.

?DUP — duplicate only if non-zero

: ?DUP   DUP IF DUP THEN ;

Duplicates TOS, tests it with IF (consuming the duplicate), then duplicates again only if non-zero. If TOS is 0, the stack is unchanged.

U. — print an unsigned number

: U.  ( u -- )
  10 /MOD
  ?DUP IF U. THEN
  CHAR 0 + EMIT ;

Divide by 10: remainder is the low digit, quotient is the rest. If the quotient is non-zero, recurse to print higher digits first. Then print the remainder digit by adding its value to CHAR 0 (ASCII 48).

U.(42) call tree

U.(42)
  10 /MOD → rem=2, quot=4
  quot ≠ 0 → recurse:
    U.(4)
      10 /MOD → rem=4, quot=0
      quot = 0 → no recurse
      print '4'
  print '2'
output: 42

. — print a signed number

: .  ( n -- )
  DUP 0 < IF NEGATE CHAR - EMIT THEN
  U. ;

Tests the sign first. If negative: negate it and print a minus sign, leaving the positive absolute value. Then pass to U. for the digits.

These four words are not in the kernel — they live in your Forth source. This is the Forth way: start with a small kernel of machine-code primitives, then build everything else as defined words. NEGATE didn't need new assembly; it needed only SWAP and -.

Chapter Program

\ Chapter 9 example — Numbers on Screen

: NEGATE   0 SWAP - ;
: ?DUP     DUP IF DUP THEN ;

: U.  ( u -- )
  10 /MOD
  ?DUP IF U. THEN
  CHAR 0 + EMIT ;

: .  ( n -- )
  DUP 0 < IF NEGATE CHAR - EMIT THEN
  U. ;

: MAIN
  6 7 *   . CR     \ 42
  -3 4 *  . CR     \ -12
  1000    . CR     \ 1000
  -1      . CR ;   \ -1

MAIN HALT
