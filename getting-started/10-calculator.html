THE CALCULATOR 1

Running Forever

Every program so far has run to completion and stopped. A calculator is different — it should keep running until you decide to quit. That calls for a loop with no natural exit: BEGIN … AGAIN.

BEGIN … AGAIN

Loop unconditionally. Execution jumps back to BEGIN every time AGAIN is reached. The only way out is EXIT from inside the loop body (or HALT).

: FOREVER
  BEGIN
    KEY EMIT
  AGAIN ;

FOREVER reads a key and echoes it, then loops back immediately. It never returns. BEGIN marks the branch target; AGAIN emits an unconditional backward branch.

BEGIN … flag UNTIL

BEGIN … flag UNTIL

Loop until the flag on TOS is non-zero (true). UNTIL pops the flag and branches back to BEGIN if it is zero; falls through if non-zero.

\ count up from 0, stop when TOS = 5
: COUNT5
  0
  BEGIN
    DUP . CR
    1 +
    DUP 5 =
  UNTIL
  DROP ;

BEGIN…AGAIN and BEGIN…UNTIL compile to the same branch primitives as IF…THEN — just pointed backwards. No new kernel code is needed.

Reverse Polish Notation

Illustration Two columns side by side. Left column "Ordinary": "3 + 4 =" with an equals key circled. Right column "RPN": "3 ENTER 4 ENTER +" with no equals key. The stack is drawn as a column of blocks beside the RPN column, growing as each value is entered.

Forth's stack is already an RPN calculator. You push numbers, apply operators, and read results — all without an "equals" key. The challenge is writing the loop that reads keys and decides what to do with them.

Our calculator uses single-digit entry (0–9) to keep the input code simple. Type a digit to push it; type an operator to compute.

THE CALCULATOR 2

DIGIT? — Test and Classify a Key

The main loop needs to ask: "is this key a digit, or an operator?" DIGIT? answers that question and leaves the key on the stack so the caller can use it either way. 

: DIGIT?  ( c -- c flag )
  DUP CHAR 0 < IF 0 EXIT THEN
  DUP CHAR 9 > IF 0 EXIT THEN
  -1 ;

The EXIT pattern returns early from a word. If the character is below '0', we push 0 and exit immediately — no need to test further. If it's above '9', same treatment. Only if both tests fail do we fall through to push -1 (true).

EXIT  ( -- )

Return from the current word immediately, as if ; had been reached. Useful for early-return patterns inside IF … THEN.

DISPATCH — Handle an Operator Key

: DISPATCH  ( key -- )
  DUP CHAR + = IF DROP +              . CR EXIT THEN
  DUP CHAR - = IF DROP -              . CR EXIT THEN
  DUP CHAR * = IF DROP *              . CR EXIT THEN
  DUP CHAR / = IF DROP /MOD SWAP DROP . CR EXIT THEN
  DUP 13 =     IF DROP DUP            . CR EXIT THEN
  DUP CHAR C = IF DROP DROP                  EXIT THEN
  DROP ;

Each line tests one key character and — if it matches — drops the key, performs the operation, and EXITs. The pattern is a chain of DUP = IF … EXIT THEN tests; exactly one branch runs, or the final DROP silently discards an unrecognised key.

CALC — The Main Loop

: CALC
  BEGIN
    KEY DIGIT? IF
      DUP EMIT 32 EMIT   \ echo digit and a space
      CHAR 0 -
    ELSE
      DISPATCH
    THEN
  AGAIN ;

Wait for a key. If it's a digit, subtract CHAR 0 (48) to convert the ASCII code to a numeric value and push it. Otherwise hand the key character to DISPATCH. Then loop forever.

Forth Words

BEGIN … AGAIN
Infinite loop — no exit unless EXIT/HALT
BEGIN … UNTIL
Loop until flag is non-zero
EXIT
( -- ) return from current word early

Patterns

DUP n = IF … EXIT THEN — dispatch on value

DUP X < IF 0 EXIT THEN — early-return test

CHAR 0 - converts ASCII digit to number

Concepts

BEGIN…AGAIN is an unconditional backward branch

AGAIN and UNTIL reuse the same branch primitives as IF/ELSE/THEN

EXIT enables clean multi-exit word design

RPN calculators are a natural fit for a stack machine
