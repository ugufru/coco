SAY SOMETHING 1

One Line Isn't Enough

In Chapter 1, you learned to put characters on the screen with EMIT. You printed letters, used CHAR to avoid memorizing ASCII codes, and even did a little arithmetic to produce a character. But every character appeared on the same row — the first one.

What if you want to print something on the second line? Or fill the screen with a title card? For that, you need a way to move the cursor to the start of the next row. In Forth, that word is CR.

CR  ( -- )

Move the cursor to the start of the next row. If the cursor is already at the start of a row, it moves to the start of the next row — just like pressing Return on a typewriter.

Stack effect: ( -- ) — CR takes nothing and leaves nothing. It just moves the cursor.

Here's a small program using CR: 

CHAR H EMIT CHAR I EMIT CR
CHAR ! EMIT HALT
Your CoCo's screen:
HI
!

The HI appears on row 1. Then CR moves the cursor to the start of row 2. Then ! appears there. Two lines, exactly as planned.

The CoCo's screen has 32 columns and 16 rows. CR advances the cursor to the start of the next row no matter which column you're currently in. At the bottom of the screen it wraps back to the top.

Words That Do a Complete Job

Illustration A factory conveyor belt. Each station (a little CoCo robot) does one job — one prints a letter, the next prints another, the last one pulls a big CR lever. The finished product rolls off the end of the belt: a complete line of text.

Once you have CR, a useful pattern emerges: you can write words whose job is to print a complete line and then move the cursor to the next row. The word takes care of everything — you just call it.

Here's a word called TITLE that prints COLOR FORTH and ends the line: 

: TITLE
  CHAR C EMIT
  CHAR O EMIT
  CHAR L EMIT
  CHAR O EMIT
  CHAR R EMIT
  32 EMIT
  CHAR F EMIT
  CHAR O EMIT
  CHAR R EMIT
  CHAR T EMIT
  CHAR H EMIT
  CR ;

TITLE HALT
Your CoCo's screen:
COLOR FORTH

The word TITLE handles eleven characters and a CR. The caller — the line TITLE HALT — doesn't need to know any of that. It just says: print the title. This is how Forth programs grow: one well-named word at a time.

SAY SOMETHING 2

Words That Call Other Words

Here's where things start to get interesting. A Forth word can call any other word that's already been defined — including words you wrote. This is how you build up a vocabulary.

Suppose you want to print a horizontal rule — a row of dashes. You could write eleven CHAR - EMIT calls in a row. Or you could define a word that prints a single dash, and then call it eleven times: 

: DASH  CHAR - EMIT ;

: RULE
  DASH DASH DASH DASH DASH DASH DASH DASH DASH DASH DASH
  CR ;

DASH is a tiny word. It does exactly one thing. RULE calls it eleven times and finishes with CR. Each word does its job cleanly. Notice that DASH has to be defined before RULE — in Forth, you can only call words that have already been defined.

Forth programmers define a lot of small words and compose them into bigger ones. The goal is for every word to read like plain English when you say it out loud: "print the title, print the rule, stop."

The Chapter 2 Program

Putting it all together — a title card, two lines, clean and organized: 

: DASH  CHAR - EMIT ;

: RULE
  DASH DASH DASH DASH DASH DASH DASH DASH DASH DASH DASH
  CR ;

: TITLE
  CHAR C EMIT CHAR O EMIT CHAR L EMIT CHAR O EMIT CHAR R EMIT
  32 EMIT
  CHAR F EMIT CHAR O EMIT CHAR R EMIT CHAR T EMIT CHAR H EMIT
  CR ;

TITLE RULE HALT
Your CoCo's screen:
COLOR FORTH
-----------

The top-level program is just three words: TITLE, RULE, HALT. You could read it to someone who knows nothing about Forth and they'd understand what it does. That's the goal.

Illustration CoCo character standing back, arms crossed, admiring the screen. Screen shows "COLOR FORTH" on one line and "-----------" beneath it. Expression: satisfied, like an artist stepping back from a finished painting.

What Happens When You Call a Word

When the program reaches TITLE, the inner interpreter jumps into the TITLE definition and runs every word inside it in sequence. When it hits ; (EXIT), it jumps back and continues with RULE. When RULE calls DASH, the same thing happens — jump in, run, return.

The machine keeps track of where to return using the return stack — a second stack, separate from the data stack you learned about in Chapter 1. You never touch it directly (not yet, anyway). It just works.

Two Stacks

Data stack — holds the numbers your program is working with. This is the one you push and pop.

Return stack — holds the addresses the interpreter needs to come back to after a word finishes. Managed automatically.

SAY SOMETHING 3

Tracing Through the Program

Here's what happens when the interpreter runs TITLE RULE HALT, step by step:

Word What happens Screen
TITLE jumps into TITLE definition
CHAR C EMIT prints C C
… EMIT × 10 prints OLOR FORTH COLOR FORTH
CR cursor moves to row 2 COLOR FORTH
(cursor here)
RULE jumps into RULE
DASH × 11 prints 11 dashes (each calls DASH) -----------
CR cursor moves to row 3
HALT program stops

Notice that the data stack is empty the entire time — no numbers are involved except briefly inside each EMIT. Words that only do output are like that: they push a value and immediately consume it.

Do-It-Yourself
Illustration CoCo character rolling up sleeves, determined expression, green screen glowing in the background waiting for input.
  1. Add a third line to the title card — your name, or anything you like. Define a new word for it and call it after RULE.
  2. Write a word called BANNER that calls RULE, then TITLE, then RULE again. What does the screen look like?
  3. What happens if you call CR before printing anything? Try it and explain the result.
  4. Write a word called SPACE that emits a single space character (32 EMIT). Then use SPACE in a word that prints two words with a gap between them.
