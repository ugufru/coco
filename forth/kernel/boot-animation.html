At power-on, the SAM maps BASIC ROM + cartridge space to $8000-$FEFF. The lower 32K is RAM. BASIC starts and waits for commands. $FF00-$FFFF is always I/O (PIA, SAM, vectors).
Step 1
CLOADM loads DECB records
XRoar auto-types CLOADM. BASIC reads four DECB data blocks into the lower 32K (the only RAM visible while ROM occupies $8000+): variables at $0050, bootstrap at $0E00, staged kernel at $1000, and the application at $2000.

The kernel was assembled at $E000 but fc.py remapped it to $1000 so BASIC can load it. App code is limited to ~24K ($2000-$7FFF) by this CLOADM constraint.
Step 2
Bootstrap enables all-RAM
CLOADM jumps to the exec address $0E00 (bootstrap).

ORCC #$50 — mask interrupts
STA $FFDF — set SAM TY bit: all-RAM mode

ROM disappears. $8000-$FEFF is now writable RAM. The $8000-$DDFF region becomes available for runtime data (but was never loaded by CLOADM).
Step 3
Copy $1000 → $E000
The bootstrap copies the staged kernel to its final location in the ROM 2 region:

LDX #$1000 — source
LDY #$E000 — destination
LDD ,X++ / STD ,Y++ — word copy loop
CMPY #KERN_END / BLO

~1.1K copied in ~570 iterations.
Step 4
JMP START — kernel runs
JMP START — the kernel initialises PIA, clears the screen, sets up stacks (S=$E000, U=$DE00), and enters the Forth application thread at $2000 via NEXT.

The app calls rg-init, which points VRAM to $0600 — harmlessly overwriting the dead bootstrap and staged kernel.
